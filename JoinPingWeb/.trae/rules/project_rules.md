
1. 技术规范：使用 Vue 2 选项式 API + 纯 JavaScript，禁止 TypeScript。对接 SpringBoot 3.4.7 后端，所有数据交互必须通过 API 调用，每处 API 需注释标注服务功能。

2. 代码质量：强制要求详细注释，复杂逻辑需先用注释列出多种方案并说明疑惑点。注重组件复用，封装通用组件如分页器。代码嵌套不超过 4 层，主动降低模块耦合度。CSS 所有颜色都用纯色数值给到最多，细节由用户自己调整。

3. 项目要求：仿虎扑评分 UI，针对 iPhone 12 Pro 设计竖屏适配布局。底部导航包含"首页"、"分类"、"我的"三个标签页。页面层级：话题列表页 → 话题详情(含主体列表) → 主体详情(含一级评论) → 评论详情(含二级评论)。

4. 数据规范：使用模拟数据开发，用注释描述完整数据结构。关键业务逻辑留白并用 TODO 标记。禁止生成任何测试代码或测试数据，只输出前端代码文件。

5. 其他：生成代码后不要自动运行，确保代码简洁易懂，技术选型以简单直接为原则。
